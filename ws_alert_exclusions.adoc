= 警示排除
:allow-uri-read: 


[role="lead"]
警報排除功能可讓您禁止針對特定檔案類型、使用者和目錄路徑產生警報。使用此功能可以排除已知且預期的活動，從而減少誤報，因為這些活動可能與警報模式相符。

導覽至 *Workload Security > Policies*，然後選擇 *Alert Exclusions* 標籤。

警報排除項目依租戶配置，包括以下內容：

. 檔案類型：抑制特定檔案類型的 File Tampering 警報<<allowed-file-types,允許的檔案類型>>
. 使用者：隱藏由選定使用者觸發的警示。
. 路徑：抑制所選目錄路徑上活動的警示（最多四個目錄層級，從交接路徑開始）。


使用者排除清單支援自動補全功能，管理員可以選擇要從警示中排除的使用者。

目錄排除最多支援四級目錄路徑，從 junction 路徑開始。

可以將個別使用者和目錄路徑新增至個別排除清單或從中移除。每個清單中的每個項目都會包含新增至清單的時間和新增者資訊，這些資訊將用於稽核目的。



== 管理排除項目和稽核

* 管理員可以隨時新增或移除檔案副檔名、個別使用者或路徑。
* 每個排除項目都會記錄設定該項目的管理員姓名和設定日期。此資訊將保留以供稽核。

== 允許的檔案類型

可以將檔案副檔名新增至_允許的檔案類型_清單中，以防止不必要的警示。

選擇 *+File Type* 可將檔案類型新增至警報允許的檔案類型清單中。

image:ws_allowed_file_types.png["新增允許的檔案類型"]

一旦新增到_允許的檔案類型_清單中，就不會針對該允許的檔案類型產生檔案篡改攻擊警報。請注意，_允許的檔案類型_策略僅適用於檔案篡改偵測。

例如，如果名為 _test.txt_ 的檔案被重新命名為 _test.txt.abc_，而 Workload Security 由於 _.abc_ 副檔名偵測到勒索軟體攻擊，則可以將 _.abc_ 副檔名新增至 _allowed file types_ 清單中。新增至清單後，系統將不再針對具有 _.abc_ 副檔名的檔案產生勒索軟體攻擊。

允許的檔案類型可以是精確匹配（例如「.abc」），也可以是表達式（例如 ++.*type++、++.type*++ 或 ++.*type*++）。不支援 .a*c 和 .p*f 類型的表達式。
